# 财报日 IV 铁鹰期权自动交易系统架构设计与业务方案

## 一、 完整实战业务方案

### 1. 应用场景
本系统专注于美股财报季的隐含波动率（IV）套利。利用财报公布前 IV 极度膨胀，财报公布后 IV 瞬间回归（IV Crush）的特性，在财报前一交易日临近收盘时构建铁鹰（Iron Condor）组合，在财报后次日开盘或持有至到期平仓获取权利金收益。

### 2. 过滤引擎 (Filter Engine)
为保障胜率，系统实施严格的标的与市场环境过滤机制：
- **流动性过滤**：剔除期权买卖价差（Bid-Ask Spread）过大、成交量/持仓量不足的标的。
- **IV 等级过滤**：IV Percentile (IVP) 需处于历史高位（例如 > 80%），且预期财报波动率显著高于历史实际波动率。
- **避险过滤**：避开宏观经济数据发布日、大盘极度恐慌状态（VIX 暴增）或个股近期存在并购重组等异常事件。

### 3. 退出机制 (Exit Strategy)
预设严格的止盈止损与离场规则，杜绝“扛单”：
- **开盘 IV Crush 止盈**：财报公布后首个交易日开盘，IV 暴降且组合实现预期利润的 50% - 80% 时，立即触发止盈。
- **硬性止损保护**：标的股价出现超预期单边暴涨或暴跌，触及保护翼的盈亏平衡点时，无条件平仓止损。
- **时间止损**：若无特殊情况，通常不持有过周末，避免周末不可控事件风险。

### 4. 极简运维 (Simplified DevOps)
- 无状态设计，重启可恢复。
- 依赖轻量级定时任务触发（如财报日前一天美东时间 15:45 触发建仓逻辑）。
- 集中化的日志与钉钉/飞书/Telegram 监控告警，确保交易全链路透明。

## 二、 设计模式与系统架构

本系统采用领域驱动设计（DDD）理念，结合经典设计模式提升系统的可扩展性与健壮性：

### 1. 模板方法模式 (Template Method)
**场景**：交易引擎生命周期管理。
**实现**：定义 `IronCondorTradingTask` 的核心执行骨架（如：获取候选池 -> 运行过滤链 -> 构建组合 -> 执行下单 -> 监控退出），子类只需实现具体步骤的具体业务逻辑，确保整体交易流程不被破坏。

### 2. 责任链模式 (Chain of Responsibility)
**场景**：标的过滤引擎。
**实现**：将流动性过滤、IV 过滤、事件避险过滤封装为独立的 Filter 节点。候选标的按顺序经过责任链，一旦被某个节点否决即刻丢弃，极大地提升了过滤逻辑的解耦与可插拔性。

### 3. 策略模式 (Strategy)
**场景**：不同的退出与止盈止损机制。
**实现**：定义 `ExitStrategy` 接口，根据当前市场状态或个股特性，动态注入 `IVCrushTakeProfitStrategy` (快速止盈策略) 或 `MaxLossStopStrategy` (最大回撤止损策略)。

### 4. 简单工厂/抽象工厂模式 (Factory)
**场景**：期权合约组合构建与订单生成。
**实现**：使用 `OrderFactory` 统一封装底层 Broker API 的差异，根据配置计算出的行权价（Strikes）安全、快速地构建出合规的 Iron Condor 复合订单对象。

### 5. 观察者模式 (Observer)
**场景**：事件监控与运维告警。
**实现**：交易生命周期中（如建仓成功、止损触发、系统异常）发布事件，`NotificationObserver` 监听这些事件并负责将消息推送到相应的运维通道（如飞书/钉钉），实现核心交易逻辑与告警逻辑的彻底解耦。
